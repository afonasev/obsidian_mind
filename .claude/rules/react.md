## React-конвенции

React 19, функциональные компоненты, TypeScript strict.

### Хуки

- Хуки — только на верхнем уровне компонента или другого хука. Biome ловит это правилом `useHookAtTopLevel`.
- Массив зависимостей `useEffect` / `useMemo` / `useCallback` — полный и честный. Biome ловит `useExhaustiveDependencies`. Не подавляем правило ради «оптимизации» — это всегда баг в ожидании.
- Не используем `useEffect` для синхронизации с пропсами или для деривируемого состояния. Если значение производится из пропсов / стора — считаем его прямо в рендере или через `useMemo`.
- Подписки на внешние источники — через `useSyncExternalStore` или готовый хук библиотеки (`useStore` из `zustand`).
- Кастомные хуки именуются `use…` и возвращают стабильный тип.

### Компоненты

- Без `React.FC` — он навязывает `children` и плохо стыкуется с дженериками. Прописываем сигнатуру руками: `export function Foo(props: FooProps): JSX.Element`.
- Возвращаемый тип компонента — `JSX.Element` явно (под strict так чище).
- Пропсы — отдельный тип / интерфейс рядом с компонентом: `interface FooProps { … }` или `type FooProps = { … }`. Inline-объекты в сигнатуре запрещаем — мешают переиспользованию и читаемости.
- Композиция важнее наследования. Большой компонент режем на мелкие; общую логику выносим в кастомный хук.
- Один компонент — один файл; имя файла совпадает с именем компонента (`CloudNode.tsx`).

### Стилизация

- Только CSS Modules: `Foo.module.css` рядом с `Foo.tsx`, импорт `import styles from "./Foo.module.css"`.
- Inline-стили (`style={{ … }}`) — только когда значение динамическое и его нельзя выразить через класс (например, координаты узла, ширина прогресс-бара). В любом другом случае — класс.
- Без Tailwind / styled-components / CSS-in-JS — это сознательное решение, см. `docs/decisions/`.

### Прочее

- Не используем `dangerouslySetInnerHTML`. Если возникнет необходимость — выносим в отдельный компонент с явной валидацией входа и комментарием-WHY.
- Event handler-ы именуем `handleSomething` внутри компонента; на пропсе соответствующий callback — `onSomething`.
- `key` для списков — стабильный id (например, `node.id`), не индекс.
- Тесты компонентов — через `@testing-library/react` + `user-event`; см. `.claude/rules/tests.md`.
