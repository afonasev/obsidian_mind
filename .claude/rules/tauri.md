## Tauri-конвенции (Rust + tauri.conf.json)

На текущем этапе в `src-tauri/` нет кастомных команд — только пустой `tauri::Builder::default()`. Правила ниже работают как контракт для будущих PR, которые добавят работу с файлами.

### Команды (`#[tauri::command]`)

- API — узкое. Каждая команда отвечает за одну операцию, аргументы типизированы конкретными структурами (не `serde_json::Value`).
- Валидация входа — на входе функции, до любых сайд-эффектов. Пути к файлам приводим к каноничному виду и проверяем на побег из разрешённого корня.
- Ошибки — типизированные, через свой `enum AppError: Serialize + thiserror::Error`. Не возвращаем `String` или `anyhow::Error` во фронт — фронт не сможет различать кейсы.
- Возврат — `Result<T, AppError>`, не `Option`. `Option::None` без причины — хуже, чем явный enum.

### Rust-стиль

- Никаких `unwrap()` / `expect()` в продакшен-пути. Заменяем на `?` с осмысленной ошибкой. Исключение — `main.rs` для финальной точки запуска приложения (там уже есть `.expect(...)` от scaffold-а Tauri — его не трогаем без причины).
- Никаких блоков `unsafe` — если кажется, что нужно, обсуждаем с разработчиком.
- Логирование — через `log::*` (уже подключён `tauri-plugin-log` в debug-сборке). Не пишем в `stdout` напрямую.
- Async — через рантайм Tauri; команды могут быть `async fn`, но не запускаем свой Tokio-рантайм без причины.

### `tauri.conf.json`

- `app.security.csp` — заполняем как только появится сетевой ввод/вывод. На текущем этапе допускаем `null` (статика из `dist/`).
- Capabilities в `src-tauri/capabilities/` — минимально необходимые. Добавляем плагин или разрешение — добавляем capability с явным `windows` / `permissions`-списком. Никаких wildcard'ов вроде `fs:*`.
- `bundle.targets` оставляем `"all"`, пока не сложилась релизная матрица.
- Изменения в `tauri.conf.json` всегда сопровождаем правкой `docs/architecture.md`, если меняется поведение IPC / окна / прав.

### IPC

- Данные между фронтом и Rust проходят сериализацию JSON. Не передаём через команды большие бинарные блобы (>1 MB) — для них Tauri даёт streaming-API.
- На фронте оборачиваем `invoke<T>(…)` в типизированный wrapper, чтобы тип результата не утекал в `any`.
- Любая команда, читающая ФС, обязана принимать путь как параметр, а не брать его из глобального состояния процесса. Это упрощает тесты и аудит.

### Тесты

- Юнит-тесты Rust — `#[cfg(test)]` рядом с кодом, через `cargo test --manifest-path src-tauri/Cargo.toml`. Интегрируем в CI отдельным change, когда появится первая команда.
- Tauri-IPC e2e — отложен, пока команд нет. Когда появятся, рассмотрим `tauri::test::mock` либо Playwright через Tauri webdriver.
