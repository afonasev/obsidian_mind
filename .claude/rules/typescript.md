## TypeScript-конвенции

Правила обязательные. Понижать строгость нельзя — `tsconfig.json` и Biome это ловят, а ревьюер уточнит почему.

### Строгий режим — контракт

- В `tsconfig.json` включены `strict`, `noUncheckedIndexedAccess`, `noImplicitOverride`, `exactOptionalPropertyTypes`. Эти флаги не понижаем.
- `tsc --noEmit` обязан проходить без ошибок и предупреждений в любой момент времени.

### Запрещено

- `any` — Biome ловит `noExplicitAny`. Для внешних/неизвестных данных используем `unknown` и сужаем через type guards.
- Non-null assertion `!` — Biome ловит `noNonNullAssertion`. Если значение может быть `null` / `undefined`, делаем явную проверку.
- `as Foo` без обоснования. Каст допустим только когда: (а) сужение от `unknown` после валидации; (б) бренд-каст для номинальных типов. В обоих случаях рядом обязателен комментарий-WHY.
- `// @ts-ignore` / `// @ts-expect-error` без комментария-причины в той же строке. Bare-ignore запрещён.
- `// biome-ignore …` без поясняющего комментария после двоеточия — линтер сам это требует, не обходим.

### Импорты

- Тип-импорты — только через `import type { … }`. `verbatimModuleSyntax` это требует.
- Группировку и сортировку делает Biome (`organizeImports`). Руками не правим.
- Из `@/…`-алиасов или относительных путей — на усмотрение модуля, но в пределах одного файла единообразно.

### Именование

- `PascalCase` — типы, интерфейсы, классы, React-компоненты, enum'ы (если появятся; предпочитаем union из литералов).
- `camelCase` — переменные, функции, методы, поля, пропсы.
- `UPPER_SNAKE_CASE` — экспортируемые константы модуля (`DB_NAME`, `DEFAULT_SAVE_DELAY_MS`).
- Файлы — `kebab-case.ts` (`debounced-saver.ts`); компоненты-файлы — `PascalCase.tsx` (`CloudNode.tsx`).

### Типы данных

- `unknown` вместо `any` для входа из внешних источников (IDB, IPC, JSON). Валидируем — потом сужаем.
- `readonly` для полей и массивов, которые не должны мутировать (см. `domain/types.ts` — там это уже норма).
- Предпочитаем `interface` для публичных контрактов и `type` для union/intersection. Не смешиваем в одном файле без причины.
- Номинальные типы можно делать через `type NodeId = string` + alias — это уже наш паттерн в `src/domain/types.ts`.
- `never` — для исчерпывающих switch'ей: явная функция `assertNever(x: never): never` лучше «само получится».

### Прочее

- `null` vs `undefined`: используем то, что уже принято в соседнем коде. В IDB-репозитории `loadGraph` возвращает `Graph | null` — придерживаемся этого.
- Возвращаемый тип публичных функций пишем явно. Внутри тел — выводим, не шумим.
