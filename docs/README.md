# Obsidian Mind — техническая документация

Этот каталог — расширенное описание проекта для разработчиков и LLM-агентов. Дополняет корневой `README.md` (что это и как запустить) и `CLAUDE.md` (как работать в этом репозитории).

## Состав

| Документ | О чём |
| --- | --- |
| [architecture.md](./architecture.md) | Слои фронта (`domain → persistence → store → components`), потоки данных, роль Tauri |
| [frontend.md](./frontend.md) | UI-компоненты, стор, стили, интеграция с `@xyflow/react` |
| [storage.md](./storage.md) | Схема IndexedDB, формат `Node` / `Edge`, дебаунс автосохранения, sanitize |
| [decisions/](./decisions/) | Ключевые архитектурные решения с обоснованием (в формате «что / зачем / цена / альтернативы / дата») |

## Аудитория

- Разработчики, входящие в проект.
- LLM-агенты, работающие с кодовой базой.
- Все, кому нужно понять *что* делает приложение и *как оно устроено* без чтения всего кода.

## Что не входит

- Быстрый старт и команды разработки — см. корневой [`README.md`](../README.md).
- Правила разработки (стиль кода, паттерны React, конвенции тестов) — см. [`CLAUDE.md`](../CLAUDE.md) и [`.claude/rules/`](../.claude/rules/).
- Планы и спецификации текущих изменений — см. [`openspec/`](../openspec/).

## Как поддерживать в актуальном состоянии

- Документы здесь описывают **текущее** состояние проекта, не план.
- При изменении схемы / архитектуры / UI обновляйте соответствующий файл в том же PR — триггеры перечислены в [`.claude/rules/docs.md`](../.claude/rules/docs.md).
- Если расхождение между кодом и документом замечено — поправить документ или удалить устаревший раздел.
