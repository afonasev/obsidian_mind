# GuitarForEveryone — техническая документация

Этот каталог содержит развёрнутое описание проекта: фичи, архитектуру, схему БД, ключевые технические решения. Это дополнение к корневому `README.md` (быстрый старт) и `CLAUDE.md` (правила разработки).

## Состав

| Документ | О чём |
| --- | --- |
| [features.md](./features.md) | Пользовательские возможности и сценарии |
| [architecture.md](./architecture.md) | Слои Clean Architecture, поток данных, правила зависимостей |
| [database.md](./database.md) | Схема БД, таблицы, индексы, FTS5 и миграции |
| [api.md](./api.md) | REST-эндпоинты по модулям |
| [frontend.md](./frontend.md) | Структура статики, i18n, темы, страница песни |
| [deployment.md](./deployment.md) | Деплой, раскладка на VPS, systemd, Caddy, бэкапы |
| [decisions.md](./decisions.md) | Ключевые технические решения и их обоснование |

## Аудитория

- Разработчики, входящие в проект.
- Авторы LLM-агентов, работающих с кодовой базой.
- Все, кому нужно понять *что* делает приложение и *как оно устроено* без чтения всего кода.

## Что не входит

- Быстрый старт и команды разработки — см. корневой `README.md`.
- Правила разработки (стиль кода, паттерны FastAPI, конвенции тестов) — см. `CLAUDE.md` и `.claude/rules/`.
- Планы и спецификации текущих изменений — см. `openspec/`.

## Как поддерживать в актуальном состоянии

- Документы здесь описывают **текущее** состояние проекта, не план.
- При изменении фичи / схемы БД / архитектуры обновляйте соответствующий файл в том же PR.
- Если расхождение между кодом и документом замечено — поправить документ или удалить устаревший раздел.
