# Тёмная тема: CSS-токены + localStorage + xyflow `colorMode` (2026-06-06)

## Что

- Вводится слой дизайн-токенов: глобальный `src/theme.css` (импорт в `main.tsx`) — `:root { --color-… }` (светлая палитра) и `[data-theme="dark"] { --color-… }` (тёмная). Все 8 CSS-модулей переведены с хексов на `var(--…)`; имена токенов семантические (фон, текст, граница, акцент, тени, состояния).
- Активная тема — атрибут `data-theme` (`light` | `dark`) на `<html>`. Выбор хранится в `localStorage` под одним ключом; при первом запуске тема берётся из `prefers-color-scheme`, дальше приоритет у сохранённого выбора.
- Логика темы в `src/theme/`: чистые функции (`resolveInitialTheme`, `persistTheme`, `applyTheme`) и хук `useTheme` (`useSyncExternalStore`, отдаёт `theme` + `toggle`).
- Переключатель темы и триггер справки — `<ControlButton>` в горизонтальной панели `<Controls orientation="horizontal">` канваса. `<ReactFlow colorMode={theme}>` синхронизирует канвас с темой.
- Анти-вспышка: инлайн-скрипт в `<head>` `index.html` проставляет `data-theme` до первой отрисовки.

## Зачем

- **CSS-переменные, а не «тёмные» классы на компонентах.** Переменные каскадируются и работают с CSS Modules без сборочной магии; одна точка переопределения (`[data-theme="dark"]`) вместо дублирующих наборов классов и риска рассинхрона. Tailwind / CSS-in-JS запрещены конвенциями (`react.md`).
- **`localStorage`, а не IDB-стор.** Тема — глобальная UI-преференция, не часть графа. `localStorage` синхронен и читается до первого кадра; IDB асинхронен → гарантированная вспышка светлой темы на старте.
- **Анти-вспышка инлайн-скриптом.** `data-theme` нужно выставить до загрузки React-бандла; единственное надёжное место — синхронный скрипт в `<head>`. React при монтировании лишь синхронизируется с уже выставленным атрибутом (`useSyncExternalStore`), без `useEffect`-синхронизации производного состояния.
- **Штатные средства `@xyflow/react`** (`colorMode`, `<Controls orientation>`, `<ControlButton>`) — без новых зависимостей и без ручной перекраски рёбер/контролов канваса.
- **Системная тема только как однократный посев.** Живая подписка на смену темы ОС — Non-Goal: лишняя сложность ради редкого сценария (YAGNI).

## Цена

- **Дублирование правила выбора темы** между инлайн-скриптом (`index.html`, вне покрытия Vitest) и `theme.ts`. Держим обе реализации тривиальными и идентичными; ключ `localStorage` и значения (`light`/`dark`) зафиксированы как контракт. Тестируемая логика живёт в `src/theme/` (100% покрытие), инлайн-скрипт — минимальный HTML.
- **`prefers-color-scheme` через `matchMedia`** отсутствует в jsdom — добавлен шим в `src/test-setup.ts` (по умолчанию светлая; тесты, которым нужна тёмная, подменяют `matchMedia` сами).
- **Перенос триггера `?`** из правого верхнего угла в панель управления — осознанный трейд-офф ради минимализма (согласовано с пользователем); диалог перепривязан к низу-слева.
- **Подбор тёмной палитры** выведен из светлых токенов вручную — не гарантирует идеальный контраст во всех узлах; правится точечно правкой `theme.css` без касания компонентов.
