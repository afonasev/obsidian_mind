# 2026-06-06 — Vault как источник правды содержимого

## Что

Источник правды содержимого базы знаний переехал из IndexedDB в **файлы выбранного vault**:

- пространства/корни → папки, узлы с непустым телом → обычные Obsidian-native `.md`-заметки;
- служебная вёрстка (набор узлов, дерево `parentId`, позиции, стиль, свёрнутость, порядок) → скрытая `.mind/` в корне vault, зеркалящая структуру (`spaces.yaml` / `space.yaml` / `root.yaml`);
- связка заметки с узлом — по `id` во frontmatter, не по имени файла;
- доступ — через порт `VaultStore` над `VaultFs` с тремя адаптерами (FS / in-memory / localStorage);
- IndexedDB сужена до настроек приложения (`mindmap` v3, только store `meta`); активное пространство — per-vault.

Детали раскладки и форматов — [`../storage.md`](../storage.md).

## Зачем

Главная цель проекта — прозрачная личная база знаний. Пока граф жил в IndexedDB, он был «вещью в себе»: не виден в Obsidian, не ищется `grep`'ом, не синхронизируется через git. С файловым источником правды приложение становится *view* над директорией: заметки — обычные `.md`, открываются и правятся в Obsidian, версионируются git'ом. Вёрстка спрятана в dot-папку `.mind/`, которую Obsidian игнорирует, поэтому база не засоряется нашими `x/y/style`.

Конкретные решения и их причины:

- **Отказ от Obsidian Canvas** — вёрстку держим в своём YAML-формате, редактируем только нашим инструментом; Canvas навязал бы чужую модель и формат.
- **Заметка несёт минимум (`id` + тело)** — координаты/стиль в `root.yaml`, чтобы не пачкать пользовательские заметки служебными полями.
- **Связка по `id`** — устойчивость к переименованию/переносу `.md` в Obsidian: имя файла — производное от текста, но истина связи — `id`.
- **Мягкое чтение** — контент (папки + `.md`) durable, `.mind/` восстановимый кэш; любая порча вёрстки → дефолты по-элементно, без краха. Потеря `.mind/` не теряет ни одной заметки.
- **Порт с in-memory/localStorage адаптерами** — чистый маппинг тестируется без реальной ФС (100% coverage честно), а web-сборка (`preview`/e2e, без Tauri) работает на localStorage-vault.

## Цена

- **Две сущности на узел-с-телом** (`root.yaml` + заметка) — разведены по ролям: вёрстка только в `root.yaml`, заметка несёт лишь `id` + тело; единый авторитет текста — `root.yaml`.
- **Потеря `.mind/` уплощает дерево** — принято: контент цел, структуру пользователь пересоберёт; `.mind/` коммитится в git вместе с vault, поэтому в норме не теряется.
- **BREAKING без миграции** — графы из IndexedDB старых версий не переносятся (у проекта нет продакшен-пользователей); работа начинается с пустого/нового vault. IDB v2→v3 удаляет stores `graph`/`workspaces`.
- **Дифф затронутых файлов** сложнее «переписать всё»: сейвер кэширует желаемый набор файлов пространства и пишет только изменённые — больше кода и тестов, но запись минимальна и точечна (требование спеки).
- **Согласование внешних правок** (перечитка с диска, усыновление новых заметок, внешние переименования) сознательно вынесено в отдельный change `vault-open-refresh`; здесь — только формат, чтение и запись.
