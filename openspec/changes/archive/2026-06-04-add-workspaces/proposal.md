## Why

Сейчас приложение держит ровно один граф mindmap — все узлы живут в общем пространстве, разделить разные темы (работа, учёба, личное) нельзя. Чтобы инструмент годился для реальной базы знаний, пользователю нужны независимые пространства, между которыми он переключается, и каждый корень mindmap должен принадлежать конкретному пространству.

## What Changes

- Появляется сущность **пространство** (workspace): именованный контейнер, владеющий собственным графом mindmap.
- Слева добавляется **сворачиваемая панель** со списком пространств (вертикальный), кнопкой `[+]` под списком и `⋮`-меню (переименовать / удалить) у каждого пространства.
- **Создание** по `[+]`: новое пространство создаётся сразу в режиме inline-редактирования имени и немедленно становится активным.
- **Удаление** через `⋮`-меню сопровождается попапом подтверждения; после удаления активного пространства активным становится соседнее по списку (если список опустел — переходим в пустое состояние).
- **Граф привязан к пространству**: на канвасе всегда отображается граф активного пространства; переключение в списке меняет видимый граф.
- **Корни нельзя создавать без активного пространства** — состояние «нет пространств» = пустой канвас с подсказкой создать пространство.
- **Старт**: открывается последнее активное пространство; **состояние сворачивания панели** запоминается между перезапусками.
- **undo/redo** ведётся отдельно для каждого пространства в рамках сессии (не персистится).
- **BREAKING**: схема IndexedDB переходит на версию `2`; ранее сохранённый одиночный граф (`graph/current`) не мигрируется — пользователь стартует с пустого списка пространств.

## Capabilities

### New Capabilities
- `workspaces`: управление пространствами — список, создание/переименование/удаление, активное пространство, сворачиваемая панель, привязка графа к пространству и переключение между ними.

### Modified Capabilities
- `local-persistence`: схема IndexedDB v2 — граф хранится по ключу `workspaceId`, добавляется хранение списка пространств, активного пространства и UI-настроек; запись `current` упраздняется.
- `mindmap-editor`: создание корневого узла разрешено только при наличии активного пространства; пустой канвас без пространств показывает подсказку вместо создания корней.

## Impact

- **Домен**: новый `src/domain/workspaces.ts` (тип `Workspace`, чистые операции create/rename/remove).
- **Persistence**: `src/persistence/db.ts` (схема v2, новые object stores), `repository.ts` (загрузка/сохранение графа по `workspaceId`, CRUD пространств, активное пространство, UI-prefs).
- **Store**: `src/store/mindmap-store.ts` — активный срез = граф активного пространства, реестр пространств + `activeId`, отдельные истории undo/redo по пространствам.
- **Компоненты**: новый `WorkspacePanel` (список, `⋮`-меню, `[+]`, попап подтверждения, сворачивание); `Canvas` рисует граф активного пространства и пустое состояние; `App` подключает панель.
- **Документация**: `README.md` (статус/структура), `docs/storage.md` (схема IDB v2), `docs/frontend.md` (панель пространств).
