## Context

Граф — это `{ nodes, edges }` с чистой раскладкой `layout()` (tidy-tree) в `domain/`. Стор держит `past`/`future` снимков графа для undo/redo и прокидывает граф в `layout()` после каждой мутации. Канвас (`Canvas.tsx`) превращает граф в узлы/рёбра React Flow; кастомный `CloudNode` рисует узел и кнопки «+».

В коде уже есть устоявшийся слой состояния-вида **вне** истории, персистимый в object store `meta`: `panelCollapsed`, `editorCollapsed`, `collapsedWorkspaceRoots`. Свёртка ветви концептуально того же рода, поэтому хранится так же — отдельно от графа, вне undo, персистится per-workspace (вариант C из explore).

## Goals / Non-Goals

**Goals:**
- Временно скрыть поддерево узла, не удаляя его, с явным визуальным признаком свёрнутости.
- Видимое дерево схлопывается аккуратно (без дыр на месте скрытого).
- Свёртка — состояние вида: вне undo/redo, переживает перезагрузку, своя у каждого пространства.
- Навигация/добавление/reveal ведут себя предсказуемо относительно скрытых узлов.

**Non-Goals:**
- Анимация сворачивания/разворачивания.
- «Свернуть всё» / «развернуть всё» одной командой, горячие клавиши для свёртки (отдельный change по YAGNI).
- Частичная свёртка (скрыть только часть детей) — сворачивается всегда всё поддерево.
- Любое изменение формата `Node`/`Edge` или версии схемы IndexedDB.

## Decisions

### Решение 1: свёртка — состояние вида вне графа и вне undo

Множество id свёрнутых узлов **активного** пространства живёт в сторе как `collapsedNodeIds: ReadonlySet<NodeId>` и персистится в `meta`. Это ставит свёртку в один ряд с `panelCollapsed`/`collapsedWorkspaceRoots`: Cmd+Z не разворачивает ветви, состояние своё у каждого пространства.

- Альтернатива A (флаг `collapsed?` на `MindNode`) дала бы нулевой плумбинг персистентности и чтение флага прямо в `layout()`, но протащила бы свёртку в undo/redo — нежелательный UX-смелл. Отклонена.
- Альтернатива B (сессионный `Map`, без персистентности) теряет свёртку на reload. Отклонена в пользу персистентного C.
- Только **активное** пространство держит свой set в state; неактивные не раскладываются и не рендерятся, их свёртка лежит в `meta` до входа. Closure-Map всех пространств (как `histories`) не нужен: toggle сохраняет сразу, вход — загружает.

### Решение 2: персистентность в `meta` без бампа схемы

Списки id свёрнутых узлов хранятся per-workspace в существующем object store `meta` под ключом `collapsedNodes:<workspaceId>` (точно как singletons `panelCollapsed` и `collapsedWorkspaceRoots`). `DB_VERSION` остаётся `2`, новый object store не нужен.

- `loadCollapsedNodes(workspaceId): Promise<readonly NodeId[]>`, `saveCollapsedNodes(workspaceId, ids)`; запись удаляется в `deleteWorkspace`.
- Загрузка — лениво при входе в пространство (рядом с `loadGraph`) и в `loadWorkspaces` для активного. Сохранение — синхронно на каждый toggle/авто-разворот (события редкие, дебаунс не нужен).
- Альтернатива — отдельный object store `collapsed`, keyed by workspaceId. Чище как модель данных, но требует бампа `DB_VERSION` до 3 + ветку upgrade. Отклонено: `meta` уже несёт ровно такие per-app/per-workspace настройки.

### Решение 3: collapse-aware `layout(graph, collapsed)`

Просто пометить узлы `hidden` в React Flow недостаточно: `subtreeRows()` всё равно зарезервирует строки под скрытое поддерево → между видимыми узлами останутся дыры. Раскладка должна знать про свёртку.

- Сигнатура: `layout(graph, collapsed: ReadonlySet<NodeId>)`. В `layoutSide`/`subtreeRows`: если узел в `collapsed`, его `rows = 1` и рекурсия в детей не выполняется. Скрытые потомки сохраняют устаревшие позиции (неважно — не рендерятся; при разворачивании `layout` пересчитает их).
- **Цена:** все вызовы `layout()` в сторе (addRoot, addChild, removeSubtree, updateText, dropNode, reparent, pasteInto и т. п.) теперь передают `collapsedNodeIds`. Это главный наклад варианта C против A. Чтобы не забыть ни один — завести в сторе локальный хелпер `relayout(graph) => layout(graph, get().collapsedNodeIds)` и звать его вместо `layout()`.

### Решение 4: `toggleCollapse` пере-раскладывает и пишет граф мимо history

Поскольку свёртка не на узле, переключение само по себе не меняет граф — но позиции видимых узлов должны схлопнуться. Поэтому `toggleCollapse(nodeId)`:

1. строит следующий `collapsedNodeIds` (add/remove `nodeId`), но только если у узла есть дети;
2. `set({ collapsedNodeIds: next, graph: layout(graph, next) })` — **без** `historyAfterPush` (вне undo). Изменение `graph` (только позиции) штатно триггерит автосейв графа;
3. сохраняет `next` через `saveCollapsedNodes(activeWorkspaceId, [...next])`;
4. если текущий `selectedNodeId` оказался в скрытом поддереве — переводит выделение на свёрнутый узел.

### Решение 5: скрытие на уровне адаптера Canvas

`Canvas` вычисляет множество скрытых id = объединение `subtreeIds(node) \ {node}` по всем узлам из `collapsedNodeIds`. `toRFNodes` отбрасывает скрытые узлы, `toRFEdges` — рёбра с хотя бы одним скрытым концом. Свёрнутый узел остаётся, но дочерние рёбра исчезают — отсюда видимый «обрыв» ветви.

### Решение 6: индикатор — стрелка + стиль узла

В `CloudNode`: рядом с кнопками «+» кнопка-переключатель `▾`(развёрнут)/`▸`(свёрнут), видна только если у узла есть дети (`hasChildren`), состояние читается из `collapsedNodeIds`. Класс `.collapsed` меняет рамку узла. У корня кнопка одна и сворачивает обе стороны.

### Решение 7: предсказуемость взаимодействий со скрытым

- **Навигация**: `findNeighbor(graph, fromId, direction, collapsed)` исключает потомков свёрнутых узлов из кандидатов.
- **Добавление ребёнка** к свёрнутому узлу (`addChildOf`, кнопка «+», Cmd/Ctrl+Enter): стор убирает родителя из `collapsedNodeIds` (+ сохраняет) перед добавлением — авто-разворот.
- **Reveal** (`revealNode`/`focusRoot`): перед центрированием стор убирает всех свёрнутых предков целевого узла из set (+ сохраняет, + relayout).

## Risks / Trade-offs

- **Рассинхрон `collapsedNodeIds` с графом при пропущенном вызове `layout`** → митигировать единым хелпером `relayout`, чтобы ни одно место мутации не звало голый `layout()`.
- **Свёрнутый узел удалён/вырезан** → его id (и id потомков) остаются в `collapsedNodeIds` мусором. Митигировать: при `removeSubtree`/`cutNode` вычистить удалённые id из set и сохранить. Для удалённого пространства запись чистится в `deleteWorkspace`.
- **Скрытый выделенный узел** при свёртке предка → перевод выделения на свёрнутый узел (Решение 4, п. 4).
- **Перетаскивание/перепривязка свёрнутого узла** → его скрытое поддерево едет с устаревшими позициями, при `dropNode`/`reparent` `relayout` оставит его листом; позиции потомков пересчитаются при развороте. Приемлемо (невидимы).
- **Вставка (paste) поддерева со свёрнутыми узлами** → клоны получают свежие id, которых нет в `collapsedNodeIds` → вставленное дерево развёрнуто. Ожидаемо.
