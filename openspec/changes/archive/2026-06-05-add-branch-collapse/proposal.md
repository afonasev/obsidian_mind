## Why

По мере роста дерева заметок канвас становится перегруженным: ветви, над которыми сейчас не работаешь, мешают видеть структуру и навигировать. Нужен способ временно скрыть поддерево, не удаляя его, и при этом ясно видеть, где именно ветвь свёрнута.

## What Changes

- Узел с детьми получает переключатель **свернуть/развернуть** на внешней грани (там же, где кнопки «+»). Виден только если у узла есть дети.
- Сворачивание скрывает **всё поддерево** узла (все нижележащие ветви, обе стороны у корня); сам узел остаётся виден.
- Свёрнутый узел визуально отличается: индикатор-стрелка `▸` (свёрнут) / `▾` (развёрнут) **и** изменённый стиль рамки узла, чтобы свёрнутость читалась и без наведения на кнопку.
- Раскладка дерева трактует свёрнутый узел как лист — видимое дерево схлопывается без «дыр» на месте скрытого поддерева.
- Клавиатурная навигация по стрелкам не заходит в скрытые узлы (свёрнутый узел навигационно — лист).
- Добавление ребёнка к свёрнутому узлу (кнопка «+», Cmd/Ctrl+Enter) **автоматически разворачивает** его — иначе новый узел был бы невидим.
- Reveal узла из панели (фокус на узел внутри свёрнутого предка) **разворачивает** свёрнутых предков, чтобы целевой узел стал виден.
- Свёрнутость — это **состояние вида**, а не графа: хранится отдельно от `nodes`/`edges`, **вне undo/redo** (Cmd+Z не разворачивает ветви), персистится per-workspace и переживает перезагрузку. Реализуется как множество id свёрнутых узлов активного пространства в сторе, с сохранением в object store `meta` под ключом на пространство.

### Modified Capabilities
- `mindmap-editor`: новое требование «Сворачивание и разворачивание ветвей»; раскладка и навигация по стрелкам учитывают свёрнутые узлы.
- `local-persistence`: в object store `meta` появляется per-workspace запись со списком id свёрнутых узлов; удаляется вместе с пространством.

## Impact

- `src/domain/layout.ts` — `layout(graph, collapsed)` трактует свёрнутый узел как лист (`subtreeRows = 1`, без рекурсии в детей).
- `src/domain/navigation.ts` — `findNeighbor` принимает множество свёрнутых и пропускает их потомков.
- `src/store/mindmap-store.ts` — состояние `collapsedNodeIds: ReadonlySet<NodeId>` (активного пространства); экшен `toggleCollapse`; прокидывание `collapsedNodeIds` во все вызовы `layout()`; загрузка/сохранение при входе/выходе из пространства; авто-разворот при `addChildOf` и при reveal; перенос выделения со скрытого узла на свёрнутый предок.
- `src/persistence/db.ts` — ключ-префикс `collapsedNodes:<workspaceId>` в `meta` (без бампа `DB_VERSION`).
- `src/persistence/repository.ts` — `loadCollapsedNodes`/`saveCollapsedNodes`; удаление записи в `deleteWorkspace`.
- `src/components/CloudNode/CloudNode.tsx` (+ CSS) — кнопка-переключатель и класс свёрнутого узла (читают `collapsedNodeIds` из стора).
- `src/components/Canvas/Canvas.tsx` — `toRFNodes`/`toRFEdges` не рендерят потомков свёрнутых узлов.
- Документация: `docs/frontend.md`, `docs/storage.md` (meta-ключ свёрнутых узлов).
- `src/domain/types.ts` и формат `Node` **не меняются**. Зависимости не меняются.
