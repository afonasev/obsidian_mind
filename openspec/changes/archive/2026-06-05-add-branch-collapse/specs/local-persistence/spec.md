## ADDED Requirements

### Requirement: Хранение свёрнутых узлов per-workspace

Приложение SHALL хранить множество id свёрнутых узлов отдельно для каждого пространства, как состояние вида (не часть графа). Хранилище SHALL использовать существующий object store `meta` с записью под ключом вида `collapsedNodes:<workspaceId>`, значение — массив id узлов (`NodeId[]`). Версия и набор object store базы при этом NOT меняются (остаётся `mindmap` v2: `graph`, `workspaces`, `meta`). Запись свёрнутых узлов SHALL удаляться при удалении пространства.

#### Scenario: Структура записи свёрнутых узлов

- **WHEN** в store `meta` присутствует запись под ключом `collapsedNodes:<workspaceId>`
- **THEN** значение записи — массив строковых id узлов (`NodeId[]`), перечисляющий свёрнутые узлы этого пространства

#### Scenario: Отсутствие записи означает «всё развёрнуто»

- **WHEN** для пространства нет записи `collapsedNodes:<workspaceId>` (свёртка никогда не применялась)
- **THEN** граф загружается без ошибок, все узлы трактуются как развёрнутые

#### Scenario: Свёртка сохраняется и восстанавливается между сессиями

- **WHEN** пользователь свернул узлы в активном пространстве, затем перезагрузил приложение
- **THEN** при входе в это пространство множество свёрнутых узлов восстанавливается из записи `collapsedNodes:<workspaceId>`

#### Scenario: Удаление пространства удаляет его свёрнутые узлы

- **WHEN** пространство удаляется
- **THEN** запись `collapsedNodes:<workspaceId>` этого пространства удаляется из store `meta` вместе с его графом
