## Why

Узел сейчас несёт только короткое имя — некуда записать развёрнутую заметку. Для базы знаний это главное: у каждой темы должно быть тело с форматированным текстом. Добавляем правую сворачиваемую панель-редактор, где тело узла пишется и читается как markdown, без отдельной кнопки сохранения.

## What Changes

- Новое опциональное поле `MindNode.body?: string` — markdown-тело узла. Формат хранения остаётся версии `2`: старые записи без `body` читаются как `undefined` (миграция не нужна).
- Правая панель-редактор `EditorPanel`, зеркальная левой `WorkspacePanel`:
  - сворачиваемая (состояние персистится в `meta` под ключом `editorPanelCollapsed`, по умолчанию развёрнута);
  - без выбранного узла — подсказка;
  - для выбранного узла: кликабельная строка «Имя родителя» (фокусирует родителя; у корня скрыта), поле-заголовок = имя узла (двусторонняя связь с инлайн-правкой на канвасе), тело узла.
- Тело узла: режим просмотра рендерит markdown (CommonMark + GFM), клик переключает в режим правки сырого markdown в `textarea`. Пустое тело показывает кликабельный плейсхолдер.
- Автосохранение тела без кнопки: правка коммитится при потере фокуса, при переключении на другой узел и через 1 секунду без ввода; дальше срабатывает существующий дебаунс-сейвер графа.
- Правка тела участвует в undo/redo наравне с правкой имени.
- Новые runtime-зависимости: `react-markdown`, `remark-gfm`.

## Capabilities

### New Capabilities

- `node-body-editor`: правая сворачиваемая панель для просмотра и редактирования markdown-тела выбранного узла, синхронизация заголовка с именем узла, автосохранение тела по триггерам.

### Modified Capabilities

- `local-persistence`: запись узла в store `graph` дополняется опциональным полем `body`; в store `meta` появляется ключ `editorPanelCollapsed`.
