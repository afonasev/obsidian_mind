## ADDED Requirements

### Requirement: Правая сворачиваемая панель-редактор

Приложение SHALL отображать справа от канваса панель-редактор тела узла. Панель MUST быть сворачиваемой: в развёрнутом виде показывает содержимое для выбранного узла, в свёрнутом — узкую полосу с кнопкой развернуть. Состояние «свёрнута/развёрнута» SHALL сохраняться между сессиями; по умолчанию панель развёрнута.

#### Scenario: Сворачивание и разворачивание панели

- **WHEN** пользователь нажимает кнопку свернуть в заголовке развёрнутой панели
- **THEN** панель сворачивается в узкую полосу с кнопкой развернуть, а содержимое (заголовок и тело) скрывается

#### Scenario: Состояние панели переживает перезапуск

- **WHEN** пользователь свернул панель, закрыл приложение и открыл снова
- **THEN** при старте панель снова свёрнута

#### Scenario: По умолчанию панель развёрнута

- **WHEN** приложение запускается впервые (сохранённого состояния панели нет)
- **THEN** правая панель отображается развёрнутой

### Requirement: Регулируемая ширина панели-редактора

В развёрнутом виде правая панель MUST поддерживать изменение ширины перетаскиванием разделителя на её левой грани (в пределах разумных минимума и максимума). Установленная ширина SHALL сохраняться между сессиями.

#### Scenario: Перетаскивание разделителя меняет ширину

- **WHEN** пользователь тянет разделитель на левой грани развёрнутой панели
- **THEN** ширина панели меняется вслед за указателем, в пределах минимума и максимума

#### Scenario: Ширина переживает перезапуск

- **WHEN** пользователь изменил ширину панели, после чего приложение перезапускается
- **THEN** при старте правая панель отображается с сохранённой шириной

### Requirement: Содержимое панели для выбранного узла

Развёрнутая панель SHALL показывать содержимое для текущего выбранного узла (`selectedNodeId`). Если ни один узел не выбран, панель MUST показывать подсказку вместо заголовка и тела. При смене выбранного узла содержимое панели SHALL переключаться на новый узел.

#### Scenario: Нет выбранного узла

- **WHEN** панель развёрнута и ни один узел не выбран
- **THEN** панель показывает подсказку (например «Выберите узел»), поля заголовка и тела не отображаются

#### Scenario: Выбор узла наполняет панель

- **WHEN** панель развёрнута и пользователь выбирает узел
- **THEN** панель показывает заголовок (имя узла) и тело этого узла

### Requirement: Ссылка на родителя

Для выбранного не-корневого узла панель SHALL показывать имя его родителя мелким кликабельным шрифтом над заголовком. Клик по имени родителя MUST выбрать родительский узел и подсветить его на канвасе. Для корневого узла (`parentId === null`) строка с именем родителя SHALL отсутствовать.

#### Scenario: Клик по имени родителя фокусирует родителя

- **WHEN** выбран не-корневой узел и пользователь кликает по строке с именем родителя в панели
- **THEN** выбранным становится родительский узел, он подсвечивается (центрируется) на канвасе, а панель переключается на родителя

#### Scenario: У корня нет строки родителя

- **WHEN** выбран корневой узел (без родителя)
- **THEN** строка с именем родителя в панели не отображается

### Requirement: Заголовок панели синхронизирован с именем узла

Поле-заголовок в панели SHALL отображать имя выбранного узла (`text`). Правка заголовка MUST немедленно менять имя узла, отражаясь на канвасе и в левой панели пространств. Обратно: правка имени узла на канвасе MUST отражаться в заголовке панели.

#### Scenario: Правка заголовка переименовывает узел

- **WHEN** пользователь меняет текст в поле-заголовке панели
- **THEN** имя выбранного узла меняется на тот же текст, и это видно на канвасе

#### Scenario: Инлайн-правка имени отражается в панели

- **WHEN** пользователь меняет имя узла инлайн на канвасе, и этот узел выбран
- **THEN** поле-заголовок в панели показывает обновлённое имя

### Requirement: Просмотр и редактирование markdown-тела

Тело узла SHALL храниться как markdown-текст. По умолчанию панель MUST показывать тело в режиме просмотра — отрендеренным markdown (CommonMark и GitHub Flavored Markdown). Клик по области тела MUST переключать её в режим правки сырого markdown в текстовом поле. Если тело пустое, в режиме просмотра MUST показываться кликабельный плейсхолдер, переключающий в режим правки.

#### Scenario: Markdown отображается отрендеренным

- **WHEN** у выбранного узла тело содержит markdown (например `# Заголовок` и список)
- **THEN** панель в режиме просмотра показывает отрендеренный markdown (заголовок и пункты списка как форматированные элементы), а не сырой текст

#### Scenario: GFM-разметка поддерживается

- **WHEN** тело узла содержит GFM-конструкцию (таблицу, чек-лист или `~~зачёркнутое~~`)
- **THEN** в режиме просмотра она рендерится по правилам GitHub Flavored Markdown

#### Scenario: Клик по телу включает правку

- **WHEN** пользователь кликает по области тела в режиме просмотра
- **THEN** область переключается в режим правки и показывает сырой markdown в редактируемом текстовом поле

#### Scenario: Пустое тело показывает плейсхолдер

- **WHEN** у выбранного узла тело пустое
- **THEN** панель показывает кликабельный плейсхолдер; клик по нему включает режим правки с пустым текстовым полем

### Requirement: Автосохранение тела без кнопки

Правка тела SHALL сохраняться автоматически, без отдельной кнопки сохранения. Несохранённый ввод MUST фиксироваться при потере фокуса текстовым полем, при переключении на другой узел и по истечении 1 секунды без ввода. Зафиксированное тело SHALL персистироваться существующим механизмом автосохранения графа.

#### Scenario: Сохранение при потере фокуса

- **WHEN** пользователь отредактировал тело и убрал фокус с текстового поля
- **THEN** новое тело фиксируется в графе выбранного узла

#### Scenario: Сохранение при переключении на другой узел

- **WHEN** пользователь редактирует тело и, не убирая фокус кнопкой, выбирает другой узел
- **THEN** правка тела первого узла фиксируется в его графе, а панель переключается на второй узел

#### Scenario: Сохранение через секунду без ввода

- **WHEN** пользователь печатает в теле и затем перестаёт печатать на 1 секунду
- **THEN** введённое тело фиксируется в графе выбранного узла без каких-либо дополнительных действий

#### Scenario: Тело переживает перезапуск

- **WHEN** пользователь задал телу узла markdown, и оно было зафиксировано, после чего приложение перезапускается
- **THEN** при выборе того же узла панель показывает прежнее тело

### Requirement: Правка тела в истории отмены

Фиксация правки тела SHALL быть отдельным шагом undo/redo, независимым от правок имени и перемещений узла. Серия последовательных правок одного тела MUST коалесцироваться в один шаг отмены.

#### Scenario: Отмена возвращает прежнее тело

- **WHEN** пользователь изменил тело узла (правка зафиксирована) и нажимает «Отменить»
- **THEN** тело узла возвращается к значению до этой правки

#### Scenario: Серия правок тела — один шаг отмены

- **WHEN** пользователь несколькими правками подряд набрал тело одного узла, затем один раз нажимает «Отменить»
- **THEN** тело возвращается к состоянию до начала этой серии правок
