## Context

Панель пространств (`WorkspacePanel`) сейчас плоская: рендерит имена пространств, активное выделено. В сторе живёт граф **только активного** пространства (`state.graph`); графы остальных лежат в IndexedDB и в память не подняты (`loadWorkspaces` грузит лишь активный граф). Корень — узел графа с `parentId === null`; их в пространстве может быть несколько.

Задача — добавить второй уровень: под каждым пространством список его корней, со сворачиванием по пространству (персист) и переходом к корню кликом (активация пространства + центрирование вьюпорта). Это затрагивает все четыре слоя: `persistence → store → components`, плюс канвас (центрирование вьюпорта живёт в React Flow).

## Goals / Non-Goals

**Goals:**
- Видеть корни всех пространств в развёрнутой панели, под именем каждого.
- Сворачивать/разворачивать список корней по каждому пространству; состояние переживает перезапуск.
- Клик по корню: активировать его пространство (если нужно), выделить корень, подвести к нему вьюпорт.
- Список корней активного пространства реактивен (создание/переименование/удаление корня видно сразу).

**Non-Goals:**
- Вложенность глубже корней (дети корней в панели не показываем).
- Drag-and-drop / переупорядочивание / переименование корней из панели — только просмотр и переход.
- Оптимизация хранения корней отдельно от графа (читаем граф целиком — YAGNI).
- Центрирование вьюпорта при обычной навигации стрелками / undo — только по явному клику в панели.

## Decisions

### Решение 1: источник корней — гибрид «живой граф + кэш»

Корни активного пространства деривятся прямо из `state.graph` в компоненте; корни неактивных читаются из кэша `rootsByWorkspace: ReadonlyMap<string, readonly PanelRoot[]>` в сторе.

```
roots(ws) = ws === activeWorkspaceId
  ? fromGraph(state.graph)            // реактивно, всегда свежо
  : rootsByWorkspace.get(ws) ?? []
```

`PanelRoot = { readonly id: NodeId; readonly text: string }` (новый тип в `domain/workspaces.ts` рядом с `Workspace`).

Карта обновляется:
- `loadWorkspaces` — заполнить для всех пространств (`repository.loadAllRoots`).
- `leaveActiveWorkspace` — записать корни покидаемого графа перед переключением (граф мог измениться, пока пространство было активным).
- `deleteWorkspace` — удалить запись.
- `createWorkspace` — новое пространство становится активным, его корни деривятся из пустого графа; запись в карте не обязательна.

**Почему так, а не единый кэш с реактивной синхронизацией:** держать корни активного и в `state.graph`, и в карте — значит дублировать данные и синхронизировать их на каждой мутации графа (подписка + диффы). Дерив из живого графа в компоненте устраняет рассинхрон by design; кэш нужен только для неактивных, которые не меняются, пока неактивны.

**Альтернатива (отклонена):** ленивая загрузка корней при первом раскрытии пространства. Меньше чтений на старте, но усложняет состояние (loading/loaded per ws) и даёт «прыжок» контента при раскрытии. При выбранном персисте свёрнутости многие пространства будут развёрнуты сразу → выгода ленивости мала.

### Решение 2: чтение корней всех пространств — `repository.loadAllRoots`

Новая функция открывает БД один раз, читает `graph`-стор через `getAllKeys` + `getAll` (ключ = id пространства, значение = `StoredGraph`), для каждого графа берёт `nodes.filter(parentId === null) → { id, text }`. Без `sanitize` (корни не зависят от целостности рёбер — берём только узлы).

```ts
async function loadAllRoots(): Promise<Map<string, readonly PanelRoot[]>>
```

Возврат `Map`, чтобы стор клал его в `rootsByWorkspace` без преобразований.

### Решение 3: центрирование вьюпорта — сигнал через стор + `fitView` в канвасе

`WorkspacePanel` находится **вне** `ReactFlowProvider`, поэтому `useReactFlow()` ей недоступен. Канвас слушает сигнал из стора:

```
state.reveal: { readonly nodeId: NodeId; readonly seq: number } | null
revealNode(nodeId): seq += 1   // монотонный счётчик-нонс, без Date.now/random
```

`CanvasInner` подписывается на `reveal` и в `useEffect` по `reveal.seq` вызывает
`fitView({ nodes: [{ id: reveal.nodeId }], maxZoom: 1, duration: 300 })` —
переиспользует уже импортированный `fitView`, который сам центрирует и зумит на узел.

Счётчик-нонс нужен, чтобы повторный клик по тому же корню (тот же `nodeId`) всё равно триггерил эффект.

Клик по корню в панели = последовательность:
1. если `ws !== activeWorkspaceId` → `await selectWorkspace(ws)` (флаш + загрузка графа);
2. `selectNode(rootId)` — выделение + запись в focus-history (как обычный выбор);
3. `revealNode(rootId)` — сигнал центрирования.

Инкапсулируем в один экшен стора `focusRoot(workspaceId, nodeId)`, чтобы панель не дирижировала последовательностью.

### Решение 4: сворачивание корней по пространству — `Set` id + META-персист

`state.collapsedWorkspaceRoots: ReadonlySet<string>` — id пространств, чьи списки корней свёрнуты. Отсутствие в наборе = развёрнуто (**дефолт — развёрнуто**). Экшен `toggleWorkspaceRoots(id)` переключает членство и пишет весь набор в META.

Хранилище: новый META-ключ `META_COLLAPSED_ROOTS_KEY = "collapsedWorkspaceRoots"`, значение — массив id (Set не сериализуется в IDB напрямую). `DB_VERSION` **не бампаем** — `meta`-стор уже существует, добавляется лишь ключ. `repository.loadCollapsedRoots(): Promise<readonly string[]>` / `saveCollapsedRoots(ids)`. При `deleteWorkspace` прунить id из набора (косметика, но без мусора).

**Почему набор свёрнутых, а не развёрнутых:** новое пространство по умолчанию развёрнуто без записи в хранилище — меньше состояния, дефолт «показать корни» бесплатен.

### Решение 5: свёрнутая панель не меняется

Ветка `collapsed` в `WorkspacePanel` (узкая полоса `»` с квадратными кнопками) уже выходит рано — корни туда просто не добавляем. Второй уровень существует только в развёрнутой панели.

## Risks / Trade-offs

- **Тайминг центрирования при смене пространства** → `selectWorkspace` грузит граф асинхронно и заменяет `state.graph`; `fitView` по id может выстрелить раньше, чем React Flow измерит новый узел. Митигация: эффект центрирования зависит и от `reveal.seq`, и от `graph` — он перезапустится после рендера нового графа; при необходимости — отложить `fitView` через `requestAnimationFrame`. Покрыть тестом cross-workspace клика.
- **Стоимость `loadAllRoots` на старте** → чтение всех графов из IDB ради списка корней. Для персонального инструмента приемлемо; при росте — вынести корни в отдельный индекс (вне scope).
- **Пустой корень (свежесозданный, без текста)** → в панели показываем плейсхолдер «Без названия», чтобы строка не была пустой и кликабельной «в никуда».
- **100% coverage** → новые ветки (toggle, reveal, cross-ws клик, пустой список, плейсхолдер) требуют тестов в `mindmap-store.test`, `WorkspacePanel.test`, `Canvas.test`, `repository.test`.
