## Context

`CloudNode` собирает класс из набора состояний (`root/selected/editing/dropTarget/collapsed`). Цвета захардкожены в `CloudNode.module.css`. `.selected` задаёт синюю рамку, `.collapsed` — серую пунктирную. `CloudNodeData` несёт только `text`; узлы для канваса строятся в `Canvas.tsx:425` (`data: { text: node.text }`). Граф-модель узла уже имеет опциональное `body?: string` (`domain/types.ts`).

## Goals / Non-Goals

**Goals:**
- Выделение свёрнутого узла читается (акцентный цвет рамки, пунктир сохранён).
- Узел с непустым телом отличается «двойной рамкой».
- Минимум правок, без нового DOM и без новых зависимостей.

**Non-Goals:**
- Токенизация цветов под темы — это зона `add-dark-theme`.
- Превью/счётчик содержимого тела, иные индикаторы помимо рамки.

## Decisions

### Фикс рамки свёрнутого выделенного узла — специфичность CSS

Причина бага: `.collapsed` и `.selected` имеют одинаковую специфичность, `.collapsed` идёт в файле позже → серый пунктир перекрывает синий. Добавляем правило `.collapsed.selected { border-color: <яркий акцент> }` (два класса = выше специфичность, побеждает независимо от порядка), `border-style: dashed` наследуется от `.collapsed`.

Альтернатива (поменять порядок правил) отклонена: хрупко, ломается при следующей перестановке; комбинированный селектор выражает намерение явно.

### Индикатор тела — `hasBody` в данных узла

`CloudNodeData` расширяется `readonly hasBody: boolean`. В `Canvas.tsx` при сборке: `hasBody: (node.body ?? "").trim() !== ""`. `CloudNode` доклеивает класс `styles.hasBody`.

Почему флаг, а не само `body`: на канвасе тело не нужно, передаём только признак — меньше данных в re-render, чище контракт.

### Индикатор тела — синеватый оттенок тени

Первый подход (внутренняя «двойная рамка» через `::after`) визуально не зашёл и убран. Вместо него `.hasBody` переопределяет `box-shadow` узла на синеватый (тон акцента `#4f80ff` → `rgba(79,128,255,…)`), с отдельным правилом для `:hover`:

```css
.hasBody { box-shadow: 0 4px 12px rgba(79, 128, 255, 0.35); }
.hasBody:hover { box-shadow: 0 6px 18px rgba(79, 128, 255, 0.45); }
```

Правила объявлены после `.node` / `.node:hover` (равная специфичность → побеждает порядок). Тень ортогональна рамке (выделение/свёрнутость) — они не конфликтуют по свойствам.

## Risks / Trade-offs

- **Параллельный `add-dark-theme` тоже правит `CloudNode.module.css`** → возможен конфликт. Митигируем порядком: вводим осмысленные цветовые значения, которые свип токенов в dark-theme заменит на `var(--…)`; в обоих change'ах правки в один файл держим локальными.
- **100% coverage:** новые ветки — `hasBody` true/false и класс `.collapsed.selected`. Покрываются компонентными/доменными тестами `CloudNode` (рендер с телом/без, выделение свёрнутого).
- **Псевдоэлемент и `border-radius`:** при `inset:4px` радиус внутренней рамки подобран на глаз; если визуально «съезжает», правится числом — низкий риск.
