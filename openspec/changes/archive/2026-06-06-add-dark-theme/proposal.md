## Why

Сейчас приложение умеет только светлую тему: ~40 цветов захардкожены хексами по 8 CSS-модулям, единого слоя дизайн-токенов нет. Тёмная тема — частое ожидание для desktop-инструмента, в котором проводят много времени; без неё интерфейс некомфортен в тёмном окружении.

## What Changes

- Вводится слой дизайн-токенов: глобальный CSS с `:root { … }` (светлая тема) и `[data-theme="dark"] { … }` (тёмная), импортируемый в `main.tsx`.
- Все захардкоженные цвета в 8 CSS-модулях заменяются на `var(--…)`.
- Добавляется переключатель темы (две позиции: светлая/тёмная) в виде `<ControlButton>` внутри штатной панели `<Controls>` от `@xyflow/react`.
- При первом запуске тема берётся из системной настройки (`prefers-color-scheme`); после первого переключения сохраняется и применяется последний выбор пользователя (`localStorage`).
- Триггер справки по горячим клавишам (`?`) переезжает из правого верхнего угла в ту же панель `<Controls>`; всплывающий диалог перепривязывается к низу-слева.
- Панель `<Controls>` становится горизонтальной (`orientation="horizontal"`).
- Канвас `@xyflow/react` получает `colorMode`, синхронизированный с текущей темой, чтобы рёбра/контролы перекрашивались штатно.
- Чтобы исключить вспышку светлой темы на старте, `data-theme` проставляется до первой отрисовки инлайн-скриптом в `<head>` `index.html`.

## Capabilities

### New Capabilities
- `theming`: выбор и хранение цветовой темы (светлая/тёмная), системное значение по умолчанию при первом запуске, переключатель в панели управления, перекраска UI и канваса.

## Impact

- **Новый код:** глобальный CSS-файл с токенами, инлайн-скрипт инициализации темы в `index.html`, хук/утилита переключения темы.
- **Затрагиваемый код:** все 8 `*.module.css` (замена цветов на токены), `src/main.tsx` (импорт CSS), `src/components/Canvas/Canvas.tsx` (`<Controls>` горизонтальная + `<ControlButton>`-кнопки + `colorMode`), `src/components/HotkeysHelp/HotkeysHelp.tsx` (триггер и позиционирование диалога), `index.html` (инлайн-скрипт).
- **Зависимости:** новых нет — используются штатные `colorMode`, `<Controls orientation>`, `<ControlButton>` из `@xyflow/react@12.3.5`.
- **Документация:** `README.md` (новая возможность), `docs/frontend.md` (тема, кнопки управления), `docs/decisions/` (новое решение: токены + localStorage + xyflow Controls), `openspec/specs/theming/`.
