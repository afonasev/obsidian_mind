## Context

Тёмная тема уже реализована (change `add-dark-theme`): есть слой дизайн-токенов `src/theme.css` (`:root` — светлая, `[data-theme="dark"]` — тёмная), все 8 CSS-модулей переведены на `var(--…)`, источник темы — `data-theme` на `<html>` + `localStorage` через `useTheme` (`useSyncExternalStore`).

Стилизация имени узла (change `add-node-style`) дала `NodeNameStyle { bold?, italic?, fontScale? }`, доменную `updateNodeStyle` (merge-патч с клампом), стор-экшен `setNodeStyle(id, patch)` (отдельный undo-шаг) и `FormatToolbar` внутри `CloudNode`, всплывающий при `isEditing` (`<NodeToolbar isVisible={isEditing}>`). Механика «клик по тулбару не уводит фокус с textarea» уже решена (`onMouseDown + preventDefault + stopPropagation`).

Эта фича добавляет цвет заливки узла поверх этой готовой базы.

## Goals / Non-Goals

**Goals:**
- Быстрый выбор цвета из заготовленной сетки прямо в тулбаре правки имени.
- Тонкая подгонка через системный picker, когда пресета мало.
- Ряд из ≤5 последних использованных цветов, чтобы не вспоминать выбранное.
- Пресеты адаптируются к теме; данные узла при смене темы не мигрируют.
- Новые дети наследуют снимок цвета родителя; старые не перекрашиваются.

**Non-Goals:**
- Авто-адаптация кастомного `#rrggbb` под тему (осознанно: «что выбрал — то и видишь»).
- Гарантия контраста текста на произвольной заливке (цена ручного выбора).
- Цвет рамки/текста отдельно от заливки — красим только фон облака.
- Палитры/темы пользователя сверх двух существующих тем.
- Применение цвета к целому поддереву задним числом.

## Decisions

### Хранить ключ пресета или сырой hex в одном поле `color`

`NodeNameStyle` получает `color?: string`. Значение — **либо** ключ пресета (например `"amber"`), **либо** `#rrggbb`. Различение при рендере — предикатом `isPresetKey(color)`: известный ключ → `var(--node-fill-${color})`, иначе — литеральный цвет.

**Почему так, а не альтернативы:** пресет-как-ключ автоматически адаптируется к теме через токены (узел хранит роль, не пиксели) — это тот же приём, что уже принят для всех 40 цветов UI. Хранить готовый hex у пресета — сломать тёмную тему. Два отдельных поля (`presetKey` + `customHex`) — лишняя сложность: значения взаимоисключающие, одно поле проще и в модели, и в MRU-списке.

`isPresetKey` опирается на фиксированный реестр ключей пресетов (единый источник — он же список для рендера сетки). Простая проверка членства; кастом всегда начинается с `#`, пересечений нет.

### Палитра пресетов — новая группа токенов в `theme.css`

Добавляем `--node-fill-<name>` для каждого пресета в обе секции (`:root` и `[data-theme="dark"]`), оттенки подобраны так, чтобы текущий `--color-fg` читался поверх в каждой теме. Реестр ключей пресетов в TS (для сетки и `isPresetKey`) перечисляет те же имена.

**Почему так:** консистентно со слоем токенов; смена темы перекрашивает заливку без участия JS. Отклонено: хранить палитру только в TS и подставлять hex инлайном — потеряли бы адаптацию к теме.

**Контраст текста:** пресеты делаем достаточно бледными (свет) / приглушёнными (тьма), чтобы `--color-fg` читался без отдельного «on-fill» токена (YAGNI). Индикатор непустого тела сейчас — оттенок тени, он ортогонален заливке и сохраняется.

### Список последних цветов — `localStorage`, вне графа и undo

MRU-список (≤5, ключи и hex вперемешку) хранится под отдельным ключом `localStorage`, по образцу `useTheme`. Чтение синхронно; запись при каждом применении цвета (unshift + dedup + срез до 5). В React — через `useSyncExternalStore` (как `useTheme`, без `useEffect`-синхронизации).

**Почему не IDB/стор графа:** это глобальная UI-преференция, общая для всех пространств, не часть графа и не должна попадать в undo/redo. localStorage синхронен — ряд недавних доступен сразу при открытии поповера.

### Наследование — снимок в домене `addChild`

`graph.addChild` читает `parent.style?.color` и кладёт его в `style.color` нового узла. Снимок на момент создания: модель иммутабельна, копируется значение, не ссылка. «Старые не перекрашиваются» выполняется само — перекраска родителя через `updateNodeStyle` создаёт новую версию только родителя.

**Почему в домене, а не в сторе:** `addChild` уже знает `parentId` и граф, это чистая операция над графом без участия React/persistence. Наследуем **только** `color`, не весь стиль (по запросу пользователя; bold/italic не копируем).

### UI: свотч в `FormatToolbar` + поповер

В `FormatToolbar` добавляется кнопка-свотч (показывает текущий цвет узла). Клик открывает поповер с: сеткой пресетов, свотчем сброса `⊘`, рядом недавних, кнопкой «свой цвет…» (`<input type="color">`). Все взаимодействия — на `onMouseDown + preventDefault + stopPropagation` (как существующие кнопки тулбара), чтобы не терять фокус textarea и не триггерить pan канваса (`nopan`). Выбор пресета/кастома/сброса → `setNodeStyle(id, { color })` (или удаление поля при сбросе) + обновление MRU.

**Сброс как удаление поля:** `setNodeStyle` сейчас сливает патч и не умеет удалять ключ. Нужна явная поддержка «убрать `color`» — патч с `color: undefined` при `exactOptionalPropertyTypes` не сотрёт ключ. Решение: в `updateNodeStyle` трактовать сброс цвета явно (например, отдельная семантика патча для очистки) — деталь реализации, но требование «сброс убирает поле» это фиксирует.

## Risks / Trade-offs

- **Кастомный hex в тёмной теме может резать глаз** → осознанный Non-Goal; пресеты остаются «безопасным» theme-aware путём, кастом — ответственность пользователя.
- **Контраст текста на произвольной заливке не гарантирован** → принято как цена ручного выбора; пресеты подобраны под читаемость.
- **Дублирование реестра пресетов (TS-ключи ↔ токены `--node-fill-*` в CSS)** → держим имена идентичными и фиксируем как контракт; при добавлении пресета правим оба места (как в `add-dark-theme` для всей палитры).
- **Сброс поля при `exactOptionalPropertyTypes`** → нужна явная семантика очистки в `updateNodeStyle`, иначе ключ `color` не сотрётся; покрываем тестом «сброс убирает поле».
- **100% coverage:** MRU-утилита и `isPresetKey` — чистый TS, тестируются честно; `<input type="color">` — нативный, тестируем через изменение значения (`user-event`/`fireEvent` смены).
- **Рассинхрон цвета узла и темы** → исключён: пресет хранит ключ, токен питает обе темы из одного источника (`theme.css`).
