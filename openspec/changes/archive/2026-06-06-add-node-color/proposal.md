## Why

Узлы карты сейчас визуально однородны — отличить ветви по смыслу можно только текстом. Цвет — самый быстрый способ группировать и навигировать по большой карте. Нужен лёгкий выбор цвета (без громоздкого color picker по умолчанию), но с возможностью точной подгонки оттенка, когда заготовленного мало.

## What Changes

- Имя-стиль узла (`NodeNameStyle`) получает опциональное поле `color` — цвет **заливки фона** облака узла. Значение — либо ключ пресета (адаптируется к теме), либо сырой `#rrggbb` (одинаков в обеих темах). Отсутствие поля = дефолтная поверхность (как сейчас). Без миграции ранее сохранённых узлов.
- В тулбаре форматирования имени (виден при инлайн-правке) появляется свотч-кружок. По клику открывается поповер: сетка заготовленных цветов, свотч сброса `⊘`, ряд из ≤5 последних использованных цветов и кнопка «свой цвет…», открывающая нативный `<input type="color">` для тонкой настройки.
- Заготовленные цвета — новая группа дизайн-токенов в `theme.css` с парами «светлая/тёмная», чтобы заливка адаптировалась к теме автоматически.
- Список последних использованных цветов (MRU, ≤5) хранится глобально в `localStorage`, вне графа и вне undo/redo (как тема и ширины панелей).
- Новый дочерний узел наследует **снимок** текущего цвета родителя в момент создания. Перекраска родителя позже НЕ перекрашивает уже созданных детей.
- Каждая смена цвета — отдельный шаг undo/redo (через существующий механизм стиля).

## Capabilities

### New Capabilities
- `node-color`: цвет заливки узла — модель (поле `color`: ключ пресета или hex), палитра заготовленных цветов (theme-aware токены), кастомный цвет через нативный picker (одинаков в обеих темах), список последних использованных (MRU ≤5, localStorage), сброс к дефолту, поведение при переключении темы.

### Modified Capabilities
- `mindmap-editor`: тулбар форматирования имени получает управляющий элемент выбора цвета (свотч + поповер); создание дочернего узла наследует снимок цвета родителя.

## Impact

- **Новый код:** группа токенов `--node-fill-*` (свет/тьма) в `src/theme.css`; модуль/утилита списка последних цветов поверх `localStorage`; поповер выбора цвета и свотч в `CloudNode`; предикат `isPresetKey` для ветвления «токен vs hex» при рендере заливки.
- **Затрагиваемый код:** `src/domain/types.ts` (`NodeNameStyle.color`), `src/domain/graph.ts` (`addChild` копирует `color` родителя; `updateNodeStyle` уже сливает патч), `src/components/CloudNode/CloudNode.tsx` + `CloudNode.module.css` (заливка из `color`, свотч в `FormatToolbar`, поповер), `src/store/mindmap-store.ts` (`setNodeStyle` уже принимает `color`; обновление MRU при выборе).
- **Зависимости:** новых нет — нативный `<input type="color">`, существующий слой токенов, существующий `setNodeStyle`-патч.
- **Документация:** `README.md` (новая возможность), `docs/frontend.md` (свотч/поповер, токены палитры, MRU), `docs/storage.md` (поле `color`, формат значения), `openspec/specs/node-color/` и дельта `mindmap-editor`.
