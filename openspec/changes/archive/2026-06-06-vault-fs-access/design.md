## Context

До этой change в `src-tauri/` нет кастомных команд — только пустой `tauri::Builder::default()`, а весь persistence идёт через IndexedDB. Чтобы перенести источник правды в файлы (последующие change `vault-storage`, `vault-open-refresh`), сперва нужен **безопасный мост к файловой системе**: узкий, типизированный, ограниченный одной директорией.

Эта change закладывает только мост и понятие активного vault. Интерпретация содержимого (структура пространств/рутов/узлов, формат `.mind/`) — в `vault-storage`. UX открытия/перечитки — в `vault-open-refresh`.

## Goals / Non-Goals

**Goals:**
- Выбор директории-vault системным диалогом и запоминание её между запусками.
- Узкий набор типизированных Rust-команд ФС, ограниченных корнем vault.
- Жёсткая защита от побега за пределы vault (canonicalize + containment).
- Типизированный фронт-wrapper без утечки `any`; предсказуемая деградация в web.

**Non-Goals:**
- Любая интерпретация содержимого файлов (формат `.mind/`, заметки) — это `vault-storage`.
- Live file-watching — осознанно отложено (синхронизация только ручной перечиткой, см. `vault-open-refresh`).
- Перенос данных из IndexedDB в файлы — миграции нет (решено ранее).
- Бинарные блобы/стриминг — работаем только с текстовыми (UTF-8) файлами.

## Decisions

### Решение 1. Активный vault держит фронт; команды принимают корень параметром

Правило `tauri.md` требует: команда, читающая ФС, **принимает путь параметром**, а не берёт его из глобального состояния процесса. Поэтому корень vault не живёт в managed-state Rust. Активный путь vault хранит фронт (в app-prefs) и передаёт его в каждую команду как `vault_root`, а целевой путь — как `rel_path` относительно корня. Это сохраняет команды чистыми, тестируемыми и аудируемыми.

### Решение 2. Узкий набор команд + типизированные ошибки

Команды (имена ориентировочные): `fs_read_dir(vault_root, rel_path)` → дерево записей `{ name, rel_path, is_dir }`; `fs_read_text(vault_root, rel_path)` → `String`; `fs_write_text(vault_root, rel_path, contents)`; `fs_create_dir(vault_root, rel_path)`; `fs_remove(vault_root, rel_path)`; `fs_rename(vault_root, from_rel, to_rel)`. Каждая возвращает `Result<T, AppError>`. `AppError` — `enum` (`Serialize + thiserror::Error`) с кейсами `PathEscape`, `NotFound`, `Io`, `NotUtf8`, `InvalidVaultRoot`, чтобы фронт различал ситуации, а не парсил строку.

### Решение 3. Confinement: canonicalize + проверка префикса

Перед любым сайд-эффектом: канонизируем `vault_root`; резолвим `rel_path` относительно него; канонизируем результат (для существующих) либо канонизируем родителя (для создаваемых); проверяем, что канонический целевой путь — потомок канонического корня. Иначе `AppError::PathEscape`. Это закрывает `..` и симлинки наружу: за симлинком, ведущим из vault, не следуем.

### Решение 4. Директорию выбирает плагин dialog; last-vault — в app-prefs

Выбор папки — через `tauri-plugin-dialog` (capability с минимальными правами, без wildcard). Выбранный абсолютный путь сохраняется как app-pref в IndexedDB (это настройка приложения, а не контент vault). На старте приложение читает last-vault и пытается его открыть; если папка исчезла — переходит в «нет vault» (поведение экрана — в `vault-open-refresh`).

### Решение 5. Деградация в web (без Tauri)

`fs-bridge` детектит наличие Tauri (`isTauri()` / отсутствие `window.__TAURI_INTERNALS__`). Без Tauri вызовы команд не делаются: wrapper сообщает «среда без ФС», приложение остаётся в состоянии «нет vault». Это сохраняет рабочими `bun run preview` и Playwright-прогон. Реальную логику маппинга на in-memory покроет `vault-storage`.

## Risks / Trade-offs

- **Побег за vault через `..`/симлинки** → canonicalize обоих путей + проверка префикса; за симлинками наружу не идём (Решение 3).
- **Web/e2e без ФС** → детект Tauri и деградация в «нет vault» (Решение 5); честное чтение/маппинг тестируется в `vault-storage` на in-memory.
- **100% coverage фронта** → `fs-bridge` тестируется моками границы IPC (граница внешняя — мок разрешён правилами); Rust-команды — `cargo test --manifest-path src-tauri/Cargo.toml`, вне vitest-coverage (как и весь `src-tauri/**`).
- **Частичная запись текстового файла при сбое** → запись через временный файл + rename там, где поддерживается; как минимум — не оставляем повреждённый `.mind/` (детальнее в `vault-storage`).
