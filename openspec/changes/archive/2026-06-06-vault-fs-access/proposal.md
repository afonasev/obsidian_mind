## Why

Источник правды сейчас — IndexedDB внутри webview: данные не видны вне приложения, их нельзя синхронизировать через git, искать сторонними инструментами или править в Obsidian. Цель проекта — быть удобным *view* над личной базой знаний в формате обычных файлов. Первый шаг — научить десктоп-обёртку (Tauri) безопасно работать с выбранной пользователем директорией-**vault**, не интерпретируя пока её содержимое.

## What Changes

- Пользователь выбирает директорию-vault системным диалогом; она становится **активным vault**.
- Путь активного vault запоминается между запусками (в app-prefs, не в самом vault); при старте приложение пытается переоткрыть его.
- Rust-сторона получает узкий набор типизированных команд работы с файлами **строго в пределах активного vault**: обход дерева каталога, чтение/запись текстового файла, создание каталога, удаление и переименование пути.
- Каждая команда принимает корень vault и путь **относительно** него; путь канонизируется и проверяется на побег за корень. Нарушение — типизированная `AppError`, не паника.
- На фронте — типизированный wrapper над `invoke`, не протекающий `any`. В web-сборке (без Tauri) wrapper деградирует предсказуемо в режим «нет vault», чтобы `preview`/e2e не падали.

## Capabilities

### New Capabilities

- `vault-access`: выбор директории-vault, запоминание активного vault между запусками и безопасный (confined) доступ к файлам внутри него из фронта через Tauri-команды.

## Impact

- **Rust**: `src-tauri/src/lib.rs` (регистрация команд), новый модуль команд ФС, `enum AppError` (thiserror + Serialize), `src-tauri/capabilities/` (плагин dialog + минимальные права), `src-tauri/tauri.conf.json`.
- **Front**: новый типизированный модуль-обёртка `src/vault/fs-bridge.ts` (`invoke<T>`-врапперы, детект Tauri); хранение last-vault в app-prefs (`src/persistence`).
- **Docs**: `docs/architecture.md` (раздел Tauri / IPC, новые команды, модель confinement), `README.md` (новое требование — выбрать vault при старте).
