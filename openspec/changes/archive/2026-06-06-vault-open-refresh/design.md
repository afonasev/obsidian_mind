## Context

`vault-access` + `vault-storage` дают доступ и формат. Эта change добавляет **пользовательский жизненный цикл** работы с vault и единственный механизм подхвата внешних правок — ручную перечитку. Решение «без live-watch, но с кнопкой refresh» принято в обсуждении явно.

## Goals / Non-Goals

**Goals:**
- Понятное состояние «нет vault» с приглашением открыть директорию.
- Открытие/смена vault и авто-переоткрытие последнего при старте.
- Кнопка «Перечитать с диска» с согласованием внешних правок по `id`.
- Активное пространство и состояние панели — per-vault.

**Non-Goals:**
- Live file-watching и авто-подхват без действия пользователя.
- Разрешение конфликтов «одновременно правили и в приложении, и на диске» сложными стратегиями слияния — применяем простую и предсказуемую политику (см. Решение 2).
- Несколько vault, открытых одновременно; история открытых vault (кроме «последнего»).

## Decisions

### Решение 1. Состояния сессии vault

`NoVault → Loaded`. Открытие vault: выбор папки → чтение через `vault-storage` → `Loaded`. Перечитка: из `Loaded` повторно читаем тот же vault. Смена vault: выбор другой папки → читаем её. При старте: если есть запомненный путь и папка существует — авто-открытие в `Loaded`, иначе `NoVault`.

### Решение 2. Перечитка = flush → re-read → reconcile по `id`

Кнопка «Перечитать с диска»:
1. синхронный `flush` отложенных записей (не теряем локальные несохранённые правки);
2. повторное чтение vault через `vault-storage`;
3. согласование по `id` с текущим состоянием.

Политика согласования (контент с диска — авторитет существования; вёрстка сопоставляется по `id`):

| Ситуация на диске | Результат |
| --- | --- |
| Появилась новая `.md` (её `id` неизвестен модели) | усыновляем как новый узел-ребёнок корня с дефолтной позицией |
| Заметка узла исчезла (`id` был, файла нет) | узел остаётся без тела, сохраняет `text` из вёрстки |
| Тело заметки изменилось | подтягиваем новое тело |
| Заметку переименовали извне (тот же `id`, другое имя файла) | связь по `id` сохраняется, обновляется указатель имени файла |
| Появилась/исчезла папка пространства или корня | список пространств/корней пересобирается из vault |

### Решение 3. Без live-watch — только явная кнопка

Файловый наблюдатель не подключаем: он тянет дебаунс, гонки с собственной записью и конфликты. Подхват внешних правок — исключительно по кнопке «Перечитать с диска». Это сознательный компромисс простоты (зафиксирован в обсуждении).

### Решение 4. Пустые состояния различаются

- **Нет vault** → приглашение «Открыть директорию-vault» (кнопка открытия).
- **Vault открыт, пространств нет** → прежняя подсказка «создайте пространство».

Это два разных экрана, чтобы пользователь понимал, на каком шаге находится.

### Решение 5. Per-vault активное пространство (панель — глобально)

Активное пространство запоминается в привязке к пути vault (app-prefs в IDB): открыл другой vault → своё активное пространство. Состояние сворачивания и ширина панели остаются **глобальными** (один ключ на приложение) — по YAGNI: per-vault раскладка панели даёт незаметную выгоду при заметной поверхности в `repository`. Решение принято при реализации; при появлении реального запроса легко перенести панель в per-vault-ключ.

### Решение 6. NoVault в web-сборке

В web/preview (на ней крутится e2e) нет нативного выбора папки. Единая модель: обе сборки стартуют в `NoVault`, пока vault не открыт. В web кнопка «Открыть директорию-vault» открывает единственный неявный localStorage-vault (без пикера) и сохраняет sentinel-путь `WEB_VAULT_PATH`, поэтому он авто-переоткрывается при следующем старте — ровно как путь в Tauri. Это делает сценарий `NoVault → открыть → refresh` полностью воспроизводимым в e2e. Перечитка повторно резолвит адаптер vault (localStorage-адаптер кэширует снимок при создании), чтобы внешние правки действительно подхватывались.

## Risks / Trade-offs

- **Конфликт «правили и там, и там»** → flush перед перечиткой + простая политика (контент с диска авторитетен для существования, вёрстка по `id`); сложного merge не делаем (Non-Goal).
- **Внешнее переименование без `id`** → не свяжется по `id`, заметка усыновится как новый узел; смягчается тем, что наши заметки всегда пишутся с `id` (`vault-storage`).
- **Дребезг состояния при частых refresh** → кнопка ручная, не авто; гонок с записью нет благодаря flush.
- **100% coverage / e2e** → reconcile-логика чистая (из `vault-storage`), тестируется на in-memory; экраны «нет vault»/refresh — `@testing-library/react`; ключевой сценарий открытия и перечитки — e2e на in-memory адаптере.
