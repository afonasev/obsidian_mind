## ADDED Requirements

### Requirement: Состояние «нет vault»

Когда активного vault нет, приложение SHALL отображать приглашение открыть директорию-vault и SHALL не показывать канвас базы знаний. Это состояние SHALL отличаться от состояния «vault открыт, но пространств нет».

#### Scenario: Первый запуск без vault

- **WHEN** приложение запускается и активного vault нет
- **THEN** отображается приглашение открыть директорию-vault с кнопкой открытия, граф не показывается

#### Scenario: Различение пустых состояний

- **WHEN** vault открыт, но в нём ещё нет пространств
- **THEN** отображается подсказка создать пространство, а не приглашение открыть vault

### Requirement: Открытие и смена vault

Приложение SHALL позволять открыть директорию как активный vault. После открытия приложение SHALL прочитать содержимое vault и отобразить его пространства, корни и граф активного пространства. Открытый vault SHALL запоминаться и переоткрываться при следующем старте.

#### Scenario: Пользователь открывает vault

- **WHEN** в состоянии «нет vault» пользователь открывает существующую директорию
- **THEN** приложение читает её содержимое и показывает пространства/корни/граф; путь запоминается

#### Scenario: Авто-открытие при старте

- **WHEN** ранее был открыт vault и его папка существует
- **THEN** при следующем старте этот vault открывается автоматически

### Requirement: Перечитка с диска с согласованием

Приложение SHALL предоставлять явное действие «Перечитать с диска», доступное при открытом vault. Действие MUST сперва синхронно сбросить (flush) отложенные записи, затем заново прочитать vault и согласовать прочитанное с текущим состоянием по `id`. Приложение SHALL не вести автоматический файловый наблюдатель — подхват внешних правок происходит только по этому действию.

#### Scenario: Подхват новой заметки, созданной извне

- **WHEN** во внешнем редакторе в папке корня создан новый `.md`, после чего пользователь нажимает «Перечитать с диска»
- **THEN** появляется новый узел-ребёнок корня с дефолтной позицией, без потери остальных узлов

#### Scenario: Подхват внешнего изменения тела

- **WHEN** тело заметки изменено извне, после чего пользователь перечитывает с диска
- **THEN** узел показывает обновлённое тело

#### Scenario: Внешнее переименование файла не рвёт связь

- **WHEN** заметка переименована извне с сохранением `id` во frontmatter, после чего пользователь перечитывает с диска
- **THEN** связь узла с телом сохраняется по `id`, узел не дублируется

#### Scenario: Заметка удалена извне

- **WHEN** `.md`-файл узла удалён извне, после чего пользователь перечитывает с диска
- **THEN** узел остаётся в структуре без тела, сохраняя свою текстовую метку
