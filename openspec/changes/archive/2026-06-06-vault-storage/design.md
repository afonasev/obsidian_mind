## Context

`vault-access` дал безопасный мост к ФС. Эта change переносит **источник правды содержимого** из IndexedDB в файлы vault и описывает формат хранения. Принцип слоёв (`domain → persistence → store → components`) сохраняется: маппинг граф↔файлы — чистый домен; доступ к диску — за портом `VaultStore`.

Опорные решения, зафиксированные в обсуждении:
- отказ от Obsidian Canvas — вёрстку держим в своём формате, редактируем только в нашем инструменте;
- тела заметок — обычные Obsidian-native `.md` (frontmatter + markdown);
- служебная вёрстка — в скрытой `.mind/` в корне vault;
- связка заметка↔узел — по `id` во frontmatter;
- синхронизация без live-watch (перечитка — в `vault-open-refresh`);
- миграции из IndexedDB нет.

## Goals / Non-Goals

**Goals:**
- Прозрачная раскладка: пространства/корни — папки, узлы-с-телом — `.md`-заметки.
- Своя вёрстка в `.mind/`, не видимая Obsidian; контент — durable truth, `.mind/` — восстановимый кэш.
- Связка по `id`, устойчивая к переименованию/переносу заметок извне.
- Мягкое чтение: любая порча вёрстки → дефолты по-элементно, без падения.
- Порт `VaultStore` с FS/in-memory адаптерами для тестируемости и web.

**Non-Goals:**
- Перечитка с диска по кнопке и согласование внешних правок — это `vault-open-refresh`.
- `[[wikilinks]]` как граф-рёбра — позже; сейчас только дерево.
- Live file-watching; миграция из IDB; двусторонняя ручная правка позиций.
- Перенос узлов между корнями/пространствами на уровне ФС-операций (как и сейчас вне scope).

## Decisions

### Решение 1. Три уровня вёрстки в `.mind/`, зеркалящие структуру

```
vault/
├─ .mind/
│  ├─ spaces.yaml                     # упорядоченный список пространств (имена папок)
│  └─ <Пространство>/
│     ├─ space.yaml                   # упорядоченный список корней + позиции корней на канвасе
│     └─ <Корень>/
│        └─ root.yaml                 # все узлы корня: id, text, parentId, x, y, style?, collapsed?, file?
├─ <Пространство>/
│  └─ <Корень>/
│     ├─ <Имя заметки>.md             # узел с телом: frontmatter `id` + markdown
│     └─ ...
```

`root.yaml` — **единственный дом структуры и вёрстки** для всех узлов корня (включая без-тельные). Запись узла с телом несёт поле `file` — текущее имя `.md`-файла. YAML выбран ради переносимости и терпимости к ручной правке.

### Решение 2. Заметка несёт минимум: `id` + тело

Frontmatter заметки содержит только `id: <NodeId>`. Остальная вёрстка (позиция, стиль, текст-метка облака) — в `root.yaml`, чтобы не засорять пользовательские заметки нашими `x/y/style`. Тело заметки — markdown после frontmatter — это значение `MindNode.body`.

### Решение 3. Текст-метка — в `root.yaml`; имя файла — производное

Авторитет текста узла (метка на облаке) — `root.yaml.nodes[].text`. Имя `.md`-файла — человекочитаемое производное от текста (sanitize запрещённых символов; при коллизии — суффикс, привязанный к `id`). При переименовании узла в приложении файл best-effort переименовывается. (Согласование внешнего переименования файла — в `vault-open-refresh`.)

### Решение 4. Связка узел↔файл по `id`

При чтении корня: для каждой записи `root.yaml` с полем `file` открываем заметку и сверяем `id` из её frontmatter. Совпал — тело привязано к узлу. Имя файла разошлось с `file`, но `id` сходится — берём по `id` (имя обновим при записи). `id` во frontmatter отсутствует/затёрт — fallback по имени файла; не нашли файла — узел остаётся **без тела**, не падаем.

### Решение 5. Мягкое чтение — дефолты по-элементно

Контент (папки + `.md`) — durable truth существования; `.mind/` — восстановимый кэш. Правила деградации:

| Что испорчено/отсутствует | Поведение |
| --- | --- |
| Битый YAML отдельной записи узла в `root.yaml` | запись скипается/берёт дефолты, остальной корень читается |
| `root.yaml` отсутствует/нечитаем | дерево корня восстанавливается из `.md`-файлов: все заметки — плоские дети корня, позиции по дефолтной раскладке |
| `space.yaml` отсутствует/нечитаем | корни = подпапки пространства, позиции по дефолту, порядок — лексикографический |
| `spaces.yaml` отсутствует/нечитаем | пространства = папки верхнего уровня vault, порядок — лексикографический |
| `file` указывает на отсутствующую заметку | узел остаётся без тела, сохраняет `text` из `root.yaml` |
| Узел в `root.yaml` без `file`, но рядом есть «лишняя» `.md` | усыновление новых заметок — в `vault-open-refresh`; при первичном чтении неизвестная заметка адаптируется как узел-ребёнок корня с дефолтной позицией |

Ни один из этих случаев не приводит к необработанной ошибке для пользователя.

### Решение 6. Порт `VaultStore` + два адаптера

`VaultStore` — интерфейс операций над vault (загрузить пространства/корни/граф корня; сохранить узел/тело/вёрстку; создать/переименовать/удалить пространство/корень/узел). Адаптеры:
- **FS** — поверх команд `vault-access`;
- **in-memory** — карта путь→содержимое, для unit-тестов и web-сборки (`preview`/e2e).

Маппинг граф↔файлы (сериализация/парсинг) — чистые функции домена над структурой «файлов», поэтому покрываются на in-memory без реальной ФС (требование 100% coverage соблюдается честно).

### Решение 7. Запись: дерево/вёрстка → `root.yaml`, тело → заметка

При мутации активного среза дебаунс-сейвер (как сейчас, 250 мс) пишет **затронутый** `root.yaml` и затронутый файл заметки. Жизненный цикл тела:
- появилось тело у узла → создаём `.md` (frontmatter `id` + тело), проставляем `file` в `root.yaml`;
- тело удалили (стало пустым) → удаляем `.md`, убираем `file`, запись узла остаётся;
- удалили узел → убираем запись из `root.yaml` и его `.md` (если был).

Запись текстовых файлов — через временный файл + rename, чтобы не оставить повреждённый `.mind/` при сбое. Перед сменой активного пространства/корня — синхронный `flush` (как в текущем сейвере).

### Решение 8. IndexedDB — только app-prefs

В IDB остаются: last vault, ширины панелей, тема, состояние сворачивания, активное пространство **per-vault** (ключ включает путь vault). Контент (графы, список пространств) из IDB уходит. `local-persistence` сужается соответственно.

## Risks / Trade-offs

- **Потеря `.mind/` уплощает дерево** → принято: контент цел, структуру пользователь пересоберёт; `.mind/` коммитится в git вместе с vault, поэтому в норме не теряется.
- **Две сущности на узел-с-телом** (`root.yaml` + заметка) → разведены по ролям: вёрстка только в `root.yaml`, заметка несёт лишь `id` + тело; единый авторитет текста — `root.yaml`.
- **Коллизии/недопустимые символы в имени файла** → sanitize + суффикс по `id`; связка всё равно по `id`, не по имени.
- **Частичная запись `.mind/`** → temp-file + rename (Решение 7); мягкое чтение переживёт даже повреждённый файл.
- **Гонка дебаунса при смене корня/пространства** → синхронный `flush` перед сменой (как сейчас).
- **Web/e2e без ФС** → in-memory адаптер; весь маппинг тестируется честно, 100% coverage.
