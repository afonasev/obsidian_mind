## Why

С появлением безопасного доступа к ФС (change `vault-access`) можно перенести источник правды содержимого из IndexedDB в файлы vault. Это и есть главная цель проекта: база знаний становится прозрачной — пространства, корни и узлы превращаются в папки и обычные `.md`-заметки, которые видны в Obsidian, ищутся `grep`'ом и синхронизируются через git. Приложение становится *view* над директорией, а не хранилищем в себе.

## What Changes

- **Источник правды содержимого** (пространства, корни, узлы, тела, дерево) — файлы vault, не IndexedDB.
- **Структура диска**: папка в корне vault = **пространство**; папка в пространстве = **корень** (одно дерево); `.md`-файл в папке корня = **узел с телом**. Узлы без тела отдельного файла не имеют.
- **Скрытая `.mind/`** в корне vault зеркалит структуру и хранит **вёрстку**: набор узлов (включая без-тельные), дерево `parentId`, позиции, стиль, свёрнутость, позиции корней, список и порядок пространств. Obsidian игнорирует dot-папки, поэтому `.mind/` не засоряет базу.
- **Связка заметки с узлом — по `id`**: заметка несёт во frontmatter только `id`, а вся вёрстка узла — в `.mind/`. Переименование/перенос файла в Obsidian не рвёт связь.
- **Мягкое чтение**: битая/отсутствующая вёрстка → дефолты по-элементно, без падения. Контент (папки + заметки) — durable truth существования; `.mind/` — восстановимый кэш вёрстки. Потеря `.mind/` не теряет ни одной заметки.
- **Порт `VaultStore`** с двумя адаптерами: FS (через `vault-access`) и in-memory (для тестов и web-сборки). Маппинг граф↔файлы — чистые функции над портом.
- **IndexedDB** остаётся только для app-prefs (last vault, ширины панелей, тема, активное пространство per-vault) — не для контента.
- **BREAKING**: данные графов из IndexedDB не мигрируются; работа начинается с пустого/нового vault.

## Capabilities

### New Capabilities

- `vault-storage`: файловая модель хранения базы знаний — раскладка пространств/корней/узлов на папки и `.md`, формат вёрстки в `.mind/`, связка заметки с узлом по `id`, мягкое чтение с дефолтами, порт `VaultStore` с FS/in-memory адаптерами.

### Modified Capabilities

- `local-persistence`: IndexedDB перестаёт хранить контент (графы, список пространств) и сохраняет только настройки приложения (last vault, UI-prefs, активное пространство per-vault).
- `workspaces`: пространство = папка верхнего уровня vault; корень = подпапка пространства; создание/переименование/удаление пространства и корня — файловые операции над vault.

## Impact

- **Domain**: новый модуль маппинга граф↔файлы (сериализация/парсинг вёрстки и заметок, sanitize имён, разбор frontmatter) — чистые функции, без React/IDB/Tauri.
- **Persistence**: порт `VaultStore` + FS-адаптер (поверх `vault-access`) + in-memory-адаптер; `repository.ts` — графы/пространства идут в vault, app-prefs остаются в IDB; `db.ts` (схема IDB сужается).
- **Store**: `mindmap-store.ts` — загрузка/сохранение активного среза через `VaultStore`; debounced-сейвер пишет затронутые `root.yaml` и файлы заметок.
- **Docs**: `docs/storage.md` (переписать под файловую модель — раскладка, формат `.mind/`, frontmatter, sanitize, мягкое чтение), `docs/architecture.md` (порт хранилища и адаптеры), новая запись в `docs/decisions/`.
