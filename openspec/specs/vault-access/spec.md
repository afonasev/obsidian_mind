# vault-access Specification

## Purpose

Доступ приложения к директории-vault на файловой системе: выбор папки системным диалогом, запоминание активного vault между запусками, ограничение всех файловых операций корнем vault и предсказуемая работа в среде без файловой системы (web-сборка).

## Requirements

### Requirement: Выбор директории-vault

Приложение SHALL позволять пользователю выбрать директорию-vault системным диалогом выбора папки. Выбранная директория MUST стать активным vault — корнем, относительно которого приложение читает и пишет файлы.

#### Scenario: Пользователь выбирает папку

- **WHEN** пользователь вызывает выбор vault и указывает существующую директорию
- **THEN** эта директория становится активным vault, её абсолютный путь сохраняется как настройка приложения

#### Scenario: Отмена выбора

- **WHEN** пользователь открывает диалог выбора и закрывает его без выбора
- **THEN** активный vault не меняется, ошибка не показывается

### Requirement: Запоминание активного vault между запусками

Путь активного vault SHALL сохраняться в настройках приложения (не внутри самого vault) и восстанавливаться при следующем запуске. Если ранее активная директория более не существует, приложение SHALL перейти в состояние «нет vault» без ошибки.

#### Scenario: Vault переоткрывается при старте

- **WHEN** пользователь выбрал vault, поработал и перезапустил приложение
- **THEN** при старте активным становится тот же vault

#### Scenario: Ранее активная папка исчезла

- **WHEN** при старте сохранённый путь vault указывает на несуществующую директорию
- **THEN** приложение переходит в состояние «нет vault», пользователю не показывается необработанная ошибка

### Requirement: Доступ к файлам ограничен корнем vault

Все файловые операции приложения SHALL выполняться строго в пределах активного vault. Каждая операция MUST принимать корень vault и путь относительно него; целевой путь MUST канонизироваться и проверяться на принадлежность корню. Попытка обратиться к пути вне корня (через `..`, абсолютный путь или симлинк наружу) SHALL отклоняться типизированной ошибкой, а не выполняться и не приводить к панике.

#### Scenario: Чтение и запись внутри vault

- **WHEN** приложение читает или пишет файл по пути внутри активного vault
- **THEN** операция выполняется, возвращается результат либо типизированная ошибка ввода-вывода

#### Scenario: Путь за пределами vault отклоняется

- **WHEN** приложение получает относительный путь, который после резолва указывает вне корня vault (например `../secret` или симлинк наружу)
- **THEN** операция отклоняется типизированной ошибкой `PathEscape`, файловая система вне vault не затрагивается

### Requirement: Предсказуемая работа без файловой системы

В среде без Tauri (web-сборка) приложение SHALL не пытаться выполнять файловые команды и SHALL оставаться в состоянии «нет vault», не падая.

#### Scenario: Запуск в web-сборке

- **WHEN** приложение запущено в браузере без Tauri
- **THEN** файловые команды не вызываются, приложение показывает состояние «нет vault» без необработанных ошибок
